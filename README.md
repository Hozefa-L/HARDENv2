# HARDEN-V2

HARDEN-V2 is a standalone public research artifact for rerunning the Phase 7 balanced smart-contract vulnerability benchmark from the included public inputs.

**Website:** https://hozefa-l.github.io/HARDENv2/

## What is in this repository

This public branch is organized for a reviewer who wants one clear experiment path rather than a full private-lab workspace. It keeps the active code, the exact public data inputs needed by the current Phase 7 configuration, and the documentation needed to understand what is included.

| Path | Purpose |
| --- | --- |
| `src/` | Active training, evaluation, preprocessing, and feature code |
| `configs/phase4.yaml` | Dataset-loading configuration used by the current experiment path |
| `configs/phase5.yaml` | Baseline-selection configuration used by the current experiment path |
| `configs/phase7_balanced.yaml` | Final public Phase 7 experiment configuration |
| `data/splits/main_benchmark/` | Train, validation, test splits and the retained `clean_default` variant manifest |
| `data/features/main_benchmark/` | Feature tables required by the public Phase 7 run |
| `data/curated/graphs/phase2_task5_20260309T161122Z/` | Graph artifacts consumed by the graph-based models |
| `reports/phase2/graph_builder_run_manifest.json` | Manifest that locates the retained graph artifact directory |
| `reports/phase3/phase3_run_manifest.json` | Manifest that locates the retained feature inputs |
| `benchmark/` | Benchmark summary and public metadata |
| `scripts/level4_analysis.py` | Aggregate analysis entrypoint for post-run summaries |

## Run the public Phase 7 experiment

Install the Python dependencies:

```bash
pip install -r requirements.txt
```

Then run the public Phase 7 configuration:

```bash
python -m src.training.run_experiments --config configs/phase7_balanced.yaml
```

The current config keeps its original device setting. If your machine does not expose that device, update `training.device` locally before starting the run.

## Outputs generated by the run

The repository ships the inputs for the public rerun path. The outputs are generated locally when you execute Phase 7.

- `checkpoints/phase7_balanced/`
- `reports/phase7_balanced/`

After the run completes, you can summarize the outputs with:

```bash
python scripts/level4_analysis.py --manifest reports/phase7_balanced/phase7_run_manifest.json
```

## Optional tooling

EtherSolve is not required for the published Phase 7 rerun path. It is only relevant if you want to revisit preprocessing workflows outside the retained reviewer artifact.

## Public scope

This branch is intended to contain the active code path plus the minimum public data and manifest surface needed to rerun Phase 7. It does not try to mirror the entire private research workspace.
